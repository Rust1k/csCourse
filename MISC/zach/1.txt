Некоторые веб-форумы имеют древовидную структуру, поэтому сообщения представляются в виде дерева. Однако сообщения обычно хранятся в базе данных как несортированный набор записей. Таким образом, при представлении сообщений пользователю древовидная структура должна быть реконструирована.

Надо реализовать логику построения дерева. Записи содержат только идентификационный номер и родительский идентификационный номер. Идентификационный номер всегда находится между 0 (включительно) и длиной списка записей (не включая). Все записи имеют родительский идентификатор ниже, чем их собственный идентификатор, за исключением корневой записи, у которой родительский идентификатор равен его собственному идентификатору.

Дерево для примера:

root (ID: 0, родительский ID: 0)
| - child1 (ID: 1, родительский ID: 0)
| | - grandchild1 (ID: 2, родительский ID: 1)
| + - grandchild2 (ID: 4, ID родителя: 1)
+ - child2 (ID: 3, ID родителя: 0)
| + - grandchild3 (ID: 6, ID родителя: 3)
+ - child3 (ID: 5, ID родителя: 0)
